> **文档角色**：策划交付给开发的唯一需求基线  
> **版本**：v1.4  
> **平台**：Windows 优先  
> **形态**：模式 A 透明叠加 + 模式 B 独立空白白板（见 CR-008）；多屏见 CR-005  
> **更新日期**：2026-07-17  
> **变更**：本轮规划多屏、清屏二次确认、独立白板（CR-005/007/008）；安装包仍搁置

## 1. 产品概述

### 1.1 一句话定义
一款 Windows 桌面应用：以**全屏透明叠加层**覆盖当前屏幕，让讲师在不打断演示的前提下，用画笔、图形、文字等工具对屏幕内容进行实时标注，并支持保存、导出与带标注截屏。

### 1.2 目标用户
- 线上/线下用电脑授课的讲师、培训师
- 需要边演示边圈点的教育类工作者
- （次要）需要在屏幕上快速示意的创作者、画家教学场景

### 1.3 核心价值
- **边讲边画**：PPT、浏览器、软件界面都在下层，标注直接画在上层
- **课堂不中断**：全局快捷键控制显示/隐藏；隐藏后鼠标可穿透，正常操作下层软件
- **结果可留存**：保存工程、导出图片、截取带标注的屏幕画面

### 1.4 本版明确不做
- macOS / Linux
- 账号体系、云同步、多人实时协作
- 激光笔动画特效、录屏、语音、课件管理
- 数位板压感深度适配（首版可不做压感；若底层易支持可作增强，非验收必选项）
- 导出 PNG 白底选项（模式 A 导出仍透明底；模式 B 自带白底画面）
- 安装包 / MSI（定版前搁置，绿色解压即可）
- 多页白板 / 课件目录

> 模式 B、多屏、清屏确认已验收。定版安装包见 **CR-009**（进行中）。

## 2. 用户场景（验收参考）

| ID | 场景 | 期望结果 |
|----|------|----------|
| S1 | 讲师打开 PPT 投屏，唤起标注层，用画笔圈重点 | 标注覆盖在 PPT 上，下层画面可见 |

## 3. 信息架构与主界面

### 3.1 窗口结构
1. **标注画布层（主窗口）**
   - 默认覆盖主显示器工作区（或全屏，见 5.1）
   - 背景透明，仅绘制用户笔迹与图形
   - 置顶（Topmost），保证盖在普通窗口之上
2. **工具栏（吸顶面板，见 CR-002）**
   - 贴主屏顶部；支持展开 / 收起（收起后仅顶部居中小入口）
   - 含：工具切换、颜色、粗细、字号、撤销/重做、清屏、保存/导出/截屏、**标注/穿透模式切换**、设置、退出、收起
3. **系统托盘（P0 必做）**
   - 应用运行期间显示托盘图标
   - 右键菜单至少：显示/隐藏标注层、退出
   - 便于穿透模式下快速找回（穿透时顶部小入口亦保留，见 CR-002）

### 3.2 两种交互状态（必须实现）

| 状态 | 笔迹可见 | 工具栏 | 能否绘制 | 鼠标是否穿透到下层 |
|------|----------|--------|----------|-------------------|
| **标注模式（Active）** | 是 | 显示 | 是 | 否（事件由本应用接收） |
| **穿透模式（Passthrough）** | **是（强制）** | 隐藏 | 否 | 是（点击落到下层应用） |

**产品约定（开发按此实现，无替代方案）：**
- **「隐藏标注层」**＝进入穿透模式：
  1. **笔迹必须仍可见**（教学对照刚需，不可做成「一并隐藏笔迹」）
  2. **完整工具栏隐藏**；但**保留顶部居中小入口**（CR-002），点击可回标注并展开工具栏
  3. **鼠标点击穿透**到下层应用；画布不接收绘制/点选
- **「显示标注层」**＝退出穿透：恢复绘制能力与工具栏（可展开）
- **工具栏收起**≠穿透：收起后仍可在标注模式下绘制（仅 UI 变小）
- 技术实现必须支持「窗口内容可见 + 点击穿透」；此为生死线能力，选型阶段必须先行验证
- 工具栏形态细节见 `docs/CR-002-工具栏吸顶展开收起.md`

## 4. 功能需求

### 4.1 工具清单（P0 = 首版必做）

| 工具 | 优先级 | 行为说明 |
|------|--------|----------|
| 画笔 | P0 | 自由曲线；按下移动绘制，松开结束一笔 |
| 荧光笔 | P0 | 同画笔路径模型；描边半透明（建议默认透明度约 40%–60%）；颜色可选 |
| 直线 | P0 | 按下起点，拖到终点松开完成；绘制中显示预览 |
| 箭头 | P0 | 同直线，终点带箭头头部 |
| 矩形 | P0 | 对角拖拽绘制；预览框随鼠标变化 |
| 圆形 | P0 | 按外接矩形或圆心-半径模型二选一，**统一用「对角拖拽外接矩形」** 降低学习成本 |
| 文字 | P0 | 点击落点出现输入框/直接输入；确认后变为画布文字对象；支持改颜色；字号为**独立三档（小/中/大）**，不与粗细联动（见 4.2） |
| 橡皮擦 | P0 | **对象擦除**：擦到某笔/某图形则删除整个对象（实现简单、可预期）。不做像素级部分擦除（可作后续） |
| 撤销 | P0 | 撤销上一步操作（绘制、删除、清屏前状态等） |
| 重做 | P0 | 重做已撤销的操作 |
| 清屏 | P0 | 清空当前画布全部对象；可撤销 |
| 取色 | P0 | 色板预设色 + 自定义颜色（系统取色器或简单色盘均可） |
| 粗细 | P0 | 至少 3 档（细/中/粗），或连续滑杆；作用于画笔/荧光笔/直线/箭头/矩形/圆形描边 |

### 4.2 样式规则
- **颜色**：全局「当前颜色」，新绘制对象使用当前颜色；已有对象首版可不支持选中后改色（降低复杂度）。
- **粗细**：全局「当前粗细」。
- **文字**：
  - 默认字体：系统无衬线（如 Microsoft YaHei UI）
  - 字号：**独立三档「小 / 中 / 大」**，与画笔粗细完全分离；工具栏单独提供字号控件
  - 支持中文输入（IME）
- **荧光笔**：必须明显半透明，与普通画笔区分

### 4.3 历史栈（撤销/重做）
- 栈容量建议 ≥ 50 步
- 清屏推入历史，可撤销恢复
- 打开新文件/新建时清空历史

### 4.4 文件与导出（P0）

| 能力 | 说明 |
|------|------|
| **保存** | 保存为应用自定义工程文件（如 `.board` / `.json`），包含所有矢量对象、样式、画布尺寸/屏幕信息 |
| **打开** | 打开已保存工程，恢复标注 |
| **导出图片** | 将当前标注层导出为 PNG，**默认且仅要求透明底**。白底选项首版不做。 |
| **截屏带标注** | 捕获当前显示器画面 + 叠加标注，导出为 PNG 到用户指定路径；默认文件名带时间戳 |

**存储建议（开发可调整，需文档化）：**
```json
{
  "version": 1,
  "canvas": { "width": 1920, "height": 1080 },
  "objects": [
    {
      "id": "...",
      "type": "pen|highlighter|line|arrow|rect|ellipse|text",
      "color": "#FF0000",
      "strokeWidth": 4,
      "opacity": 1,
      "points": [],
      "x": 0, "y": 0, "w": 0, "h": 0,
      "text": "",
      "fontSize": 24
    }
  ]
}
```

### 4.5 全局快捷键（P0）

**默认快捷键（出厂值）：**

| 功能 ID | 默认快捷键 | 功能 |
|---------|------------|------|
| `toggle_overlay` | `Ctrl + Shift + A` | 显示/隐藏标注层（切换 Active ↔ Passthrough） |
| `clear_canvas` | `Ctrl + Shift + C` | 清屏（推荐仅 Active 下生效） |
| `undo` | `Ctrl + Z` | 撤销（Active 下） |
| `redo` | `Ctrl + Y` | 重做（Active 下） |
| `save` | `Ctrl + S` | 保存 |
| `cancel` | `Esc` | 取消当前未完成绘制 / 退出文字输入（可为应用内热键，非必须全局） |

要求：
- `toggle_overlay`、`clear_canvas` 等课堂相关键必须是**全局热键**（其他应用前台时仍可用）
- `undo` / `redo` / `save`：Active 下生效；能做成全局更佳，首版至少 Active 窗口内可用

### 4.6 快捷键自定义（P0，必做）

| 项 | 要求 |
|----|------|
| 入口 | 工具栏「设置」或托盘菜单「快捷键设置」打开设置面板 |
| 可改范围 | 上表中所有**全局**功能项均可修改（至少包含：显示/隐藏、清屏；撤销/重做/保存若已注册为全局也须可改） |
| 录制方式 | 点击某项 →「按下新组合键」录制；支持 Ctrl/Shift/Alt + 字母/数字等常见组合 |
| 冲突处理 | 与本应用已占用快捷键冲突时禁止保存并提示；与系统/其他软件冲突时尽量在注册失败时提示「注册失败，请更换」 |
| 恢复默认 | 提供「恢复默认快捷键」一键还原 |
| 持久化 | 写入本地配置文件（如 `%APPDATA%/<应用名>/settings.json`），重启后保持 |
| 即时生效 | 保存后重新注册热键，无需重启应用 |

**配置示例（开发可调整）：**
```json
{
  "hotkeys": {
    "toggle_overlay": "Ctrl+Shift+A",
    "clear_canvas": "Ctrl+Shift+C",
    "undo": "Ctrl+Z",
    "redo": "Ctrl+Y",
    "save": "Ctrl+S"
  }
}
```

### 4.7 托盘与启动（P0）
- P0：**必须**提供系统托盘；菜单至少「显示/隐藏」「退出」
- P0：应用启动后进入标注模式（**推荐：启动后显示工具栏 + 标注模式**）
- P1：开机自启（不做也可）

## 5. 交互细则

### 5.1 显示器
- 首版：**仅主显示器**满覆盖即可
- 多显示器：P1（可在工具栏选「当前屏」）；本版不做不扣首版验收

### 5.2 绘制通用
- 所有拖拽类工具：按下开始、移动预览、松开提交为对象并入历史
- 画笔/荧光笔：移动采样点，过密可抽稀，保证流畅
- 绘制时工具栏不应抢焦点导致断笔（注意 hit-test）

### 5.3 文字
- 点击放置 → 弹出单行/多行输入（建议多行）→ Enter 或失焦确认；`Esc` 取消
- 已落字对象：首版可不支持二次编辑；橡皮擦可整对象删除

### 5.4 橡皮擦
- 光标改为橡皮样式或圆圈
- 与对象包围盒/路径命中检测，命中则删除并记入撤销

### 5.5 性能
- 1080p 下画笔连续书写应无明显卡顿
- 对象数量建议可支撑数百个路径点组成的笔迹而不掉帧；极端情况可提示简化

### 5.6 权限与安装
- 透明置顶窗口、全局热键：按 Windows 常规权限即可
- 截屏 API：使用 Windows 图形捕获即可，勿依赖危险驱动
- **交付形态**：首版提供**绿色解压运行**（zip 等）即可；MSI/安装向导非必做
- 需有简短使用说明（README）

## 7. 技术栈（已确认）

**首版技术栈：WPF + .NET 8（已拍板，按此实现）。**

不强制 Tauri / Electron。选型理由：透明置顶、可见+点击穿透、全局热键重注册、截屏合成在 Windows 原生栈上最稳，与产品生死线一致。

关键能力检查清单（开发开工前自证）：
1. 无边框透明置顶窗口
2. **点击穿透开启时，笔迹仍可见**（可见 + 穿透同时成立）
3. 全局热键注册、卸载与重新注册（支撑自定义快捷键）
4. 桌面截图并与画布合成导出
5. 本地读写工程文件与设置文件
6. 系统托盘图标与菜单

## 9. 验收标准（首版通过线）

开发完成「可交付」需同时满足：

1. 可在 Windows 10/11 **绿色解压运行**，UI 为中文  
2. 透明叠加标注可用，下层桌面/窗口内容可见  
3. 工具齐备：画笔、荧光笔、直线、箭头、矩形、圆形、文字（独立字号三档）、橡皮、撤销、重做、清屏、取色、粗细  
4. 可切换标注模式 / 穿透模式；穿透时能操作下层应用，且**笔迹保持可见、工具栏隐藏**  
5. 全局快捷键至少支持显示/隐藏；提供**快捷键设置页**，可修改、可恢复默认、重启后保留（S8）  
6. 支持保存与打开工程；**截屏带标注、保存工程、导出标注 PNG（透明底）均为必做**  
7. **系统托盘**可用（显示/隐藏、退出）  
8. 无阻断性崩溃；README 说明默认快捷键、如何改快捷键与基本用法

## 10. 给开发的实现备注

- **对象模型优先**：所有笔迹以矢量对象存储，便于撤销、保存、擦除整对象。  
- **坐标**：以画布客户区坐标为准；保存时记录画布宽高，打开时若分辨率变化可左上对齐或按比例缩放（首版左上对齐 + 提示即可）。  
- **Z 序**：后绘制在上；橡皮删除后重绘即可。  
- **安全**：不收集用户屏幕内容上传；截屏仅存用户本地所选路径。
