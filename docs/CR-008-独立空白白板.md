# 改版单：独立空白白板模式 B（CR-008）

> **优先级**：P0（本轮主功能之一）  
> **状态**：待开发  
> **关联**：原 PRD「模式 B」；与模式 A（透明叠加）并存；多屏见 CR-005  

---

## 1. 目标

在「屏幕透明标注（A）」之外，提供 **独立空白白板（B）**：不透出桌面，整屏（当前选中屏）为白板底，用同一套画笔/图形工具书写，适合纯板书。

---

## 2. 模式定义

| | **模式 A：屏幕标注** | **模式 B：空白白板** |
|--|----------------------|----------------------|
| 背景 | 透明，透出下层桌面/PPT | **不透明**白板底（默认纯白） |
| 绘制 | 同现有工具 | 同现有工具 |
| 穿透 | 有（笔迹可见 + 点下层） | **无穿透到桌面**（见 2.3） |
| 数据 | 独立对象列表 A | 独立对象列表 B |

### 2.1 切换入口

- 工具栏增加模式切换，建议分段或下拉：
  - **屏幕标注**（A）
  - **空白白板**（B）
- 与现有「标注模式 / 穿透模式」区分开：
  - 「屏幕标注 vs 空白白板」= **内容形态**（A/B）
  - 「可绘制 vs 穿透」= **仅 A 下** 的交互状态
- 切换 A↔B：**保留各自笔迹**，互不覆盖、不清空另一方。

### 2.2 白板外观

- 默认背景：`#FFFFFFFF`（白）。
- 可选增强（非阻塞）：工具栏「黑板」切换深色底 `#FF1E1E1E` + 浅色默认笔；没有也可，首版白底即可。
- 白板铺满 **当前选中显示器**（随 CR-005）。

### 2.3 穿透与「隐藏」在 B 下的行为

- 模式 B **不提供**「点穿白板操作桌面」（否则白板失去意义）。
- 工具栏原「穿透模式」在 B 下：
  - **隐藏/收起完整工具栏**，保留顶部小入口（与现网收起类似）；或
  - 文案改为「收起工具栏」，避免写「穿透」。
- 右键移动模式、移动工具、橡皮等在 B 下照常可用。

### 2.4 保存 / 导出 / 截屏 / 清屏

- **保存工程**：一份文件内同时存 A、B 两套对象（或分字段 `objectsOverlay` / `objectsBoard`），打开后两套都恢复；`version` 升级并兼容旧单列表文件（旧文件视为仅 A）。
- **导出 PNG**：导出 **当前模式** 画面（B 含白底+笔迹；A 仍透明底笔迹）。
- **截屏带标注**：
  - A：当前屏桌面 + A 笔迹（现逻辑，跟选中屏）；
  - B：白板底 + B 笔迹（等价于导出当前白板全屏图，可不抓桌面）。
- **清屏**：只清 **当前模式**；走 CR-007 二次确认。

### 2.5 启动

- 默认进入 **模式 A**（与现网一致），避免改变老用户习惯。

## 3. 非目标

- 多页白板、课件目录  
- 云同步、激光笔  
- 安装包  

---

## 4. 验收标准

| # | 步骤 | 期望 |
|---|------|------|
| 1 | 切到空白白板 | 全屏（当前屏）白底，不见桌面 |
| 2 | 在 B 上书写 | 工具与 A 一致，笔迹正常 |
| 3 | 切回 A | A 上原笔迹仍在；B 笔迹保留，再切回 B 可见 |
| 4 | B 下点原「穿透」 | 不点穿到桌面；可收起工具栏 |
| 5 | B 清屏确认后 | 仅 B 清空，A 不受影响 |
| 6 | 保存/打开 | A+B 都能恢复 |
| 7 | 导出/截屏 | 按当前模式结果正确 |
| 8 | 与 CR-005 | 换屏后 A/B 窗体跟屏 |

## 5. 实现备注（给开发）

- 可用同一 `MainWindow` 切背景透明/不透明 + 两套 `List<AnnotationObject>`；或 B 用独立层。
- 切换模式时更新工具栏高亮与状态文案。
- 旧 `.board`：仅一组 `objects` → 载入为 A，B 为空。
