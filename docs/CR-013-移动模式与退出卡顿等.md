> **状态**：已交付

## 1. 快捷键设置去掉「开关移动模式」

### 原因
移动模式已默认绑定 **鼠标右键短按** 切换，设置里再保留该项易重复、易误绑。

### 要求
- 从快捷键设置列表、默认配置、`HotkeyLabels` / `DefaultHotkeys` 中 **移除** `toggle_move_mode`（或当前等价 actionId）。
- 设置页不再显示、不可再录制该项。
- **右键短按切换移动模式** 行为保留，写死为产品手势（不必进可配置热键表）。
- README / 设置说明改为：移动模式用右键短按切换；工具栏「移动」工具仍可用。
- 若用户旧 `settings.json` 里仍有该键：加载时忽略并清理，勿报错。

### 验收
设置里看不到「开关移动模式」；右键短按仍能开/关移动模式。

## 2. 点击退出后桌面/鼠标卡顿数秒（缺陷）

### 现象
点「退出」后，整机桌面与鼠标明显卡顿约数秒，随后恢复。

### 根因方向（供核实）
高概率与 **全局钩子 / 热键 / 托盘** 释放顺序有关，尤其是 CR-011 若已上 **低级鼠标钩子（WH_MOUSE_LL）**：
- 退出时未及时 `UnhookWindowsHookEx`，或卸钩在阻塞/死锁路径上；
- 或退出时同步做了过重工作（大量 IO、钩子回调里清理）卡住输入队列。

亦可能：全局键盘热键未卸净、托盘图标销毁慢、进程迟迟不退出但仍半挂起。

### 要求（必须）
- 退出路径（工具栏退出、托盘退出、`Application.Shutdown`）在 **关闭窗口之前/之时尽快**：
  1. 卸载全部鼠标钩子；
  2. 卸载全部 `RegisterHotKey`；
  3. 销毁托盘图标；
  4. 再释放其它资源并退出。
- 卸钩须在 UI 线程安全、**快速**完成；禁止在钩子回调里同步 `Invoke` 重逻辑导致卡死。
- 目标：退出后 **1 秒内** 鼠标与桌面恢复流畅；无残留钩子（退出后进程应结束）。

### 验收
| # | 步骤 | 期望 |
|---|------|------|
| 1 | 正常使用含快捷键/鼠标键后点退出 | 进程退出；卡顿不明显（远短于数秒） |
| 2 | 退出后立刻移动鼠标、点桌面 | 立即流畅 |
| 3 | 任务管理器 | `ScreenAnnotator` 已结束 |

## 3. 右键打开移动模式时，自动退出橡皮/框选删除

### 要求
当 **右键短按将移动模式切为「开」** 时：
- 若当前工具是 **对象橡皮 / 笔刷橡皮 / 框选删除**（以及其它互斥的删除类工具），**自动切换**为 **移动** 工具（或上一次非删除类绘制工具；策划定案：**切到「移动」工具**，状态清晰）。
- 工具栏高亮同步到「移动」。
- 若右键是把移动模式 **关掉**：不强制改工具（可留在当前工具）。

说明：避免「移动模式已开但仍是橡皮，点到对象变成擦除」的误操作。

### 验收
在框选删除或橡皮下，右键打开移动模式 → 工具自动变为移动；可拖对象而非删除。

## 4. Delete / Backspace 删除选中对象

### 要求
- 当前存在 **一个或多个选中对象** 时：按键盘 **Delete** 或 **Backspace** → 删除全部选中对象。
- 进入撤销栈（一步可撤）。
- 无选中时：两键不吞掉系统其它用途即可（本应用内无操作）。
- 在文字输入框编辑文案时：Backspace/Delete 走输入框编辑，**不要**删掉整个文字对象（焦点在文本框时优先文本编辑）。
- 穿透模式下可不处理（或仅标注可绘时处理）；策划定：仅 **标注可绘状态（Active）** 下响应。

### 验收
选中后 Del 或退格删除；可撤销；编辑文字内容时退格只改字不删对象。
