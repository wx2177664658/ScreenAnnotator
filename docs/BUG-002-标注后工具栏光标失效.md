# 缺陷单：标注操作后工具栏箭头光标失效（BUG-002）

> **来源**：产品回归反馈（CR-003 第 1 点未彻底修好）  
> **优先级**：P0（直接影响工具栏可点性感知）  
> **状态**：待开发修复  
> **关联**：`docs/CR-003-工具栏光标布局背景.md` R1–R4  

---

## 1. 现象

- 启动后、尚未在画布上画过时：鼠标移入工具栏 → 变为普通箭头，可点击（正常）。
- **在画布上完成一次标注操作（拖拽画笔/图形等）之后**：再把鼠标移入工具栏 → **不再自动变回箭头**，点击切换功能困难或表现为失效。

---

## 2. 复现步骤（必测）

1. 启动，确认标注模式、工具栏展开。  
2. 先不画，鼠标移入工具栏 → 应为箭头。  
3. 移到画布，用画笔（或矩形等）**按下拖拽再松开**，完成一笔。  
4. 再移入工具栏 → **仍必须是箭头且按钮可点**。  
5. 重复多笔绘制后再进工具栏 → 同样必须稳定。

当前第 4 步失败。

---

## 3. 根因研判（策划基于源码，请开发核实并修）

`MainWindow.OnCanvasMouseDown` 在开始绘制时调用了：

```csharp
_canvas.CaptureMouse();
```

副作用（WPF 经典问题）：

1. 捕获期间，鼠标即使移到工具栏窗口上方，**事件仍发给画布**，工具栏往往**收不到 `MouseEnter`**。  
2. 若在工具栏区域松手，或松手后指针已停在工具栏上：捕获释放后，工具栏可能**不会再补发一次 `MouseEnter`**。  
3. 于是 `_pointerOverChrome` 仍为 `false`，`Mouse.OverrideCursor` 被清掉，主窗 `Cursor = Pen` 继续生效 → **看起来就是「画过之后进工具栏不变箭头」**。

现有 `ToolbarWindow.OnChromeMouseEnter` → `NotifyPointerOverChrome(true)` 依赖 Enter 事件，在捕获场景下不可靠，故 CR-003 修法不够。

---

## 4. 修复要求（必须）

### 4.1 目标行为（不变）

- 鼠标处于工具栏或顶部小入口客户区内：**永远**普通箭头 + 可点。  
- 离开 chrome 回到画布：恢复当前工具光标。  
- **与是否刚画过、是否 CaptureMouse 过无关。**

### 4.2 必做修复（至少完成其一组合，以稳定为准）

**A. 释放捕获后立刻校正（必做）**

在所有 `ReleaseMouseCapture` / 绘制结束路径（`OnCanvasMouseUp`、`CancelInProgress` 等）末尾调用统一方法，例如 `SyncChromePointerState()`：

- 若 `ToolbarWindow.IsMouseOver` 或 `ToolbarTabWindow.IsMouseOver` 为 true，或  
- 用当前鼠标屏幕坐标判断落在 chrome 窗矩形内  

则：`NotifyPointerOverChrome(true)` + 强制 `Mouse.OverrideCursor = Arrow`（与现有 `EnforceArrowCursor` 一致）。  
否则：`NotifyPointerOverChrome(false)` 并按规则恢复画笔光标。

**B. 不单靠 MouseEnter（强烈建议）**

增加与捕获无关的检测，任选：

- 主窗 / 画布 `PreviewMouseMove`：若坐标落入工具栏窗区域，强制箭头；或  
- 定时/在 Move 时用 `VisualTreeHelper` / Win32 `WindowFromPoint` 判断指针下是否为 chrome 窗。

**C. Capture 策略（可选优化）**

评估能否在不破坏拖拽画线的前提下减少捕获副作用（例如仅在需要时捕获，或捕获期间若进入 chrome 矩形则提前结束绘制并释放捕获）。若保留 `CaptureMouse`，则 **A+B 仍必须做**。

### 4.3 禁止

- 禁止只改「点标注模式」路径而忽略「画完再进工具栏」路径。  
- 禁止 pen 光标在 chrome 上「看起来能点但其实点到画布」。

---

## 5. 验收标准

| # | 步骤 | 期望 |
|---|------|------|
| 1 | 未绘制，进工具栏 | 箭头，可点 |
| 2 | 画一笔后，进工具栏 | **箭头，可点（本单核心）** |
| 3 | 连续画 10 笔后，进工具栏 | 仍稳定箭头可点 |
| 4 | 绘制中拖到工具栏再松开 | 结束后若指针在工具栏上 → 箭头可点；不卡死捕获 |
| 5 | 离开工具栏回画布 | 恢复画笔/当前工具光标 |
| 6 | 点「标注模式」后再画再进工具栏 | 不回归失效 |
| 7 | 穿透 / 收起小入口 | CR-002 行为不回退 |

全部通过后重新发布绿色包。

现象：一在画布上标注操作之后，再移入工具栏，箭头光标/可点就失效。

根因研判：绘制时 `_canvas.CaptureMouse()` 导致工具栏错过 `MouseEnter`，松手后指针已在工具栏上也不会补发 Enter，`_pointerOverChrome` 仍为 false，主窗 Pen 光标盖住体验。

请在 `ReleaseMouseCapture`/绘制结束时用 `IsMouseOver` 或坐标命中强制 `SyncChromePointerState`；不要只靠 MouseEnter。按文档第 5 节复现自测后再发包。
