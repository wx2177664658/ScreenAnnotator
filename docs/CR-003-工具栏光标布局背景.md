## 1. 光标：进入工具栏范围必须变为普通箭头（可点击）

### 1.1 问题

期望：鼠标**无论何时**进入工具栏（或顶部小入口）范围，光标自动变为**普通箭头（Arrow）**，以便点击按钮、切换模式。

现状反馈：多数时候尚可，但在**手动点击「标注模式」之后**，该行为失效（悬停工具栏仍可能是画笔等绘制光标，或难以点选）。

### 1.2 根因方向（供开发核实）

`MainWindow.UpdateCursor()` 在标注模式下把主窗 `Cursor` 设为 `Pen` / `IBeam` / `Cross`。工具栏为独立窗口，但：

- 主窗全屏置顶时，光标样式可能被「全局/主窗」状态干扰；
- 或点击「标注模式」后重新 `UpdateCursor()` / `Activate()`，导致悬停工具栏时仍显示绘制光标；
- 工具栏未在 `MouseEnter` / `MouseLeave` 时强制 `Cursor = Arrow`。

### 1.3 修复要求（必须）

| 规则 | 说明 |
|------|------|
| R1 | 鼠标进入 **ToolbarWindow** 任意客户区 → 立即 `Cursor = Arrow`（工具栏窗与其子控件均箭头） |
| R2 | 鼠标进入 **ToolbarTabWindow**（收起小入口）→ 同样强制箭头 |
| R3 | 鼠标离开工具栏回到标注画布 → 再恢复当前工具对应光标（画笔/文字/橡皮等） |
| R4 | 点击「标注模式」「穿透模式」、收起/展开、切换工具后，**R1–R3 仍必须成立**，不得再失效 |
| R5 | 工具栏按钮在标注模式下始终可点，不被画布抢事件 |

**建议实现**：

- `ToolbarWindow` / `ToolbarTabWindow`：`MouseEnter` 设 Arrow；必要时对根 `Border` 设 `Cursor="Arrow"`  
- `MainWindow`：仅在画布 `MouseEnter` 时 `UpdateCursor()`，**不要**在工具栏仍可见且鼠标在工具栏上时强制主窗 Pen 光标「盖过」体验  
- 若因全屏窗命中导致光标错乱：保证工具栏 `Topmost` 高于画布，且工具栏区域 hit-test 优先  

### 1.4 验收

1. 标注模式 + 已选画笔 → 移入工具栏，光标变箭头，可点任意按钮。  
2. 点击「标注模式」后再移入工具栏 → **仍是箭头且可点**（本条为回归重点）。  
3. 移出工具栏到画布 → 恢复画笔（或当前工具）光标，可正常绘制。  
4. 收起态小入口悬停 → 箭头，可点击展开。

## 2. 布局：压缩高度，次要选项点击展开

### 2.1 目标

减少吸顶栏纵向占用；主操作一行可见，颜色/字号等收纳。

### 2.2 展开态信息架构（目标结构）

**第 0 行（模式行，可较矮）**

- 标题「屏幕标注」
- 「标注模式 | 穿透模式」分段按钮
- 「收起」
- 状态文案可缩短或并入模式旁

**第 1 行（主操作行，核心）—— 画笔类 + 编辑/文件类同一行**

从左到右建议顺序（可微调，但必须同一行 / 可 Wrap 到第二行仅当宽度不够）：

1. 工具：画笔、荧光笔、直线、箭头、矩形、圆形、文字、橡皮  
2. 分隔  
3. 撤销、重做、清屏  
4. 保存、打开、导出 PNG、截屏  
5. 设置、退出  

> 用户所述「重置」按现有功能理解为 **清屏**（若另有重置入口则并入本行）。

**第 2 行起：可折叠面板（默认收起）**

用按钮打开/关闭弹出层或行内展开区，**默认全部收起**：

| 入口按钮 | 展开后内容 |
|----------|------------|
| **颜色** | 色板预设 +「自定义…」；按钮上可显示当前色小色块 |
| **粗细** | 细 / 中 / 粗（可与颜色合并为一个「样式」面板；若合并须在方案实现说明里写清） |
| **字号** | 小 / 中 / 大（主要用于文字工具） |

策划推荐：**「颜色」「粗细」「字号」三个独立折叠入口**，或 **「颜色/粗细」合并 + 「字号」独立**——开发任选，以默认高度明显变矮为准。

折叠交互：

- 点击入口 → 展开选项；再点一次或点击面板外 → 收起  
- 展开时可用下拉/弹出 `Popup`，避免把整条顶栏撑得很高  
- 当前选中值在入口上有摘要（如色块、粗细名）

### 2.3 非目标

- 不改工具种类与功能集合  
- 不做完全自定义布局  

### 2.4 验收

1. 默认展开态纵向高度明显小于当前「三行全开」版本。  
2. 工具与撤销/清屏/保存等在同一视觉主行。  
3. 颜色、字号（及粗细）默认收起，点击后可改，再收起。  
4. 1280 宽主屏下主行可接受轻度换行，但不得回到「大白块占屏 1/4」的观感。

## 3. 背景：半透明，避免纯白

### 3.1 问题

当前顶栏接近纯白（如 `#F5FFFFFF`），不美观，也偏抢画面。

### 3.2 要求

| 项 | 规格 |
|----|------|
| 工具栏底板 | **半透明深色或浅灰玻璃感**，禁止接近纯白实底 |
| 推荐 | 深色半透明：`#CC2A2A2A`～`#E6282828`（约 80%–90% 不透明深灰），白字/浅字；**或** 浅色半透明：`#B3F5F5F5`（约 70% 白，能透出下层画面） |
| 策划默认拍板 | 采用 **浅色半透明玻璃**：背景约 `#99FFFFFF`～`#B3F5F5F5`，边框 `#40FFFFFF` / 浅阴影；文字保持深色可读 |
| 小入口 Tab | 与顶栏同一套半透明风格，视觉统一 |
| 对比度 | 按钮文字、模式选中态在半透明底上仍清晰可读 |

可选增强（非必须）：轻微模糊（Acrylic）——有则加分，无也不扣本单验收。

### 3.3 验收

1. 顶栏能隐约透出下层桌面/PPT，不再是一块死白。  
2. 文字与按钮仍可读、可点。  
3. 小入口与展开栏风格一致。
