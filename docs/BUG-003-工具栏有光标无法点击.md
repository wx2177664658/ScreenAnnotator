# 缺陷单：标注后工具栏有箭头光标但无法点击（BUG-003）

> **来源**：产品回归（BUG-002 修完后的新现象）  
> **优先级**：P0  
> **状态**：待开发修复  
> **关联**：BUG-002、CR-003 R1/R5  

---

## 1. 现象

- 在画布上完成标注操作后，鼠标移入菜单栏：**光标已变为普通箭头**（BUG-002 部分生效）。
- 但在工具栏上**无法点击任何功能按钮**（切换工具、模式、撤销等均无响应）。
- 其他操作（在画布上继续画等）正常。

---

## 2. 复现步骤

1. 启动 → 标注模式 → 工具栏展开。  
2. 在画布拖拽画一笔（或矩形等）。  
3. 将鼠标移到顶部工具栏 → 光标变为箭头。  
4. 点击「画笔 / 撤销 / 穿透」等任意按钮 → **无反应**。  

第 4 步失败即为本单。

---

## 3. 根因研判（策划基于源码）

BUG-002 用 `SyncChromePointerState` + **屏幕矩形命中**（`WindowContainsScreenPoint`）把光标改成箭头。  
这只表示「指针坐标落在工具栏窗口的几何范围内」，**不表示点击真正落在工具栏 HWND 上**。

更可能的情况：

1. 绘制时主窗 `MainWindow` 全屏置顶并接收鼠标（含 `CaptureMouse` / 激活）。  
2. 松手后 **主窗 Z 序仍压在 `ToolbarWindow` 之上**（两者都是 `Topmost=True` 时，后激活的主窗常会盖住工具栏）。  
3. 指针移到顶部：主窗仍在最前接收 `PreviewMouseMove` → 几何判断「在工具栏矩形内」→ **箭头光标**；  
4. 点击却打在全屏透明主窗/画布上 → **按钮无响应**。  
5. 旁证：若真是工具栏 `IsMouseOver==true`，按钮一般能点；几何命中但 `IsMouseOver==false` 时，就是「假在工具栏上」。

次要可能（请一并排查）：

- `CaptureMouse` 未在所有路径释放，点击仍被画布吃掉（若仍捕获，也应在进 chrome 时强制 `ReleaseMouseCapture`）。

---

## 4. 修复要求（必须）

### 4.1 目标

标注后移入工具栏：**箭头 + 每个按钮可点**，与画之前一致。

### 4.2 必做：保证 chrome 窗口永远可点（Z 序）

可见的 `ToolbarWindow` / `ToolbarTabWindow` 必须在 **命中测试上位于 `MainWindow` 之上**。

建议实现（可组合）：

| 手段 | 说明 |
|------|------|
| A | 封装 `EnsureChromeAboveMain()`：对 chrome 窗 `HWND` 使用 `SetWindowPos(..., HWND_TOPMOST, ...)` 或相对主窗置顶，在以下时机调用：主窗 `Activated`、`OnCanvasMouseUp` / `FinishDrawing`、`SyncChromePointerState` 判定 over、工具栏 `Show` |
| B | 当「几何上在 chrome 内」但 `toolbar.IsMouseOver == false` 时，视为 Z 序错误，**立即** `EnsureChromeAboveMain()`，必要时短暂 `Activate` chrome（避免抢焦点导致别的问题的前提下优先保证可点） |
| C | 主窗不要在每次绘制后把自身抬到 chrome 之上；若必须 `Activate` 主窗，激活后立刻把 chrome 再抬一层 |

验收直觉：画完后用 Spy++/逻辑判断，点击顶部时 `IsMouseOver` 应为工具栏 true，而不是只有坐标落在矩形内。

### 4.3 必做：进 chrome 区域时解除画布捕获

在 `SyncChromePointerState` / `ApplyChromePointerState(true)` 中：

- 若 `_canvas.IsMouseCaptured` → `ReleaseMouseCapture()`，并结束未完成绘制（调用现有 `FinishDrawing`/`CancelInProgress` 中合适者，避免悬空状态）。

### 4.4 建议：修正「仅几何命中」的副作用

`IsPointerOverChrome()` 在仅靠矩形命中且 chrome 实际未收到鼠标时，不要只改光标；应先抬 Z 序，再以 `IsMouseOver` 或抬序后的命中为准。  
避免长期停留在「箭头但点到画布」状态。

### 4.5 回归

- BUG-001 绘制、BUG-002 箭头、CR-002 吸顶/穿透小入口、CR-003 布局背景均不回退。

## 5. 验收标准

| # | 步骤 | 期望 |
|---|------|------|
| 1 | 未画过，点工具栏按钮 | 可点 |
| 2 | 画一笔后，移入工具栏 | 箭头 |
| 3 | 画一笔后，点击工具切换/撤销/模式等 | **全部可点并生效（本单核心）** |
| 4 | 连续画多笔后再点工具栏 | 仍可点 |
| 5 | 绘制中拖到工具栏再松手 | 捕获释放；随后可点工具栏 |
| 6 | 穿透小入口 | 可点并回标注 |
